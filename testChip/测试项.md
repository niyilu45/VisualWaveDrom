# VisualWaveDrom 测试项

本文档列出 VisualWaveDrom 应用的全部功能测试场景与检查清单。

**验证方式说明**

| 标记 | 含义 |
|------|------|
| ✅ 通过 | 代码审查确认逻辑正确；和/或 Playwright/手动浏览器验证通过 |
| ❌ 失败 | 测试未通过或发现缺陷 |
| ☐ 待测 | 需联网浏览器手动确认（本环境未跑通自动化） |
| ⚠️ 部分通过 | 功能可用但存在已知限制 |

**部署说明**：`VisualWaveDrom.html` 为**单文件自包含**应用。WaveDrom 皮肤（`WaveSkin`）与库（`wavedrom.min.js`）已内嵌于 HTML `<head>` 中的两个 `<script>` 块，**无需** `inc/` 目录即可离线运行。`inc/` 保留为源码备份，升级 WaveDrom 时可编辑 `inc/` 后运行 `node testChip/embed-inc.mjs` 重新内嵌。

| 3.10 | 撤销按钮禁用态 | 初始加载 / 撤销至无可撤销时 | 「撤销」按钮半透明且不可点；有可撤销历史后恢复 | ✅ 通过 |

**防抖说明**：连续快速输入在 400ms 内视为同一次编辑，只产生一条撤销记录；停顿超过 400ms 后再输入则产生新的一条。`undo()`/`redo()` 调用前会 `flushUndoDebounce()`，故可立即撤销首键。

**快捷键实现**：`document` 捕获阶段监听 `keydown`；`Ctrl/Cmd+Z` 撤销，`Ctrl/Cmd+Shift+Z` 与 `Ctrl/Cmd+Y` 重做；`e.key.toLowerCase()` 兼容 Shift 产生的大写 `Z`；textarea 与内联编辑 overlay 均 `preventDefault` 以使用自定义栈。

| 3.5.11 | 小数列点击映射 | hscale=1.5 时点击波形行不同列 | 选中列索引与高亮区域与点击位置一致（列宽 20×1.5=30px） | ☐ 待测 |

| 3.5.6 | 撤销 hscale | 修改 hscale 后点击撤销 | JSON 与输入框、波形均回退到上一 hscale | ☐ 待测 |

**实现说明**：输入框绑定 `config.hscale`（小数 0.1–20，步进 0.1）；Enter / change / blur 触发 `pushUndoBeforeChange` 后写回 JSON 并 re-render；编辑器 JSON 变更经 `renderWaveform` → `syncHscaleInputFromJson` 反向同步。无效/NaN 输入默认 1；超出范围 clamp 至 0.1–20。列映射使用 `WAVE_UNIT_WIDTH * hscale`（默认 20px × hscale，如 hscale=1.5 则 30px/列）。WaveDrom 3 支持小数 hscale。

**已知限制**：

- 无标签重命名功能；需覆盖或删除后新建。
- 标签数量受 localStorage 配额限制（大 JSON 多标签可能触达上限）。
